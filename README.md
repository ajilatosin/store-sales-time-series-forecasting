# Store Sales Time Series Forecasting

An end-to-end machine learning solution for the Kaggle Store Sales Forecasting competition. This project develops a robust time series forecasting pipeline using advanced feature engineering and an ensemble of XGBoost and CatBoost models to predict future sales across multiple store locations and product categories.

## Project Overview

The objective of this project is to forecast daily unit sales for thousands of products sold across various stores. The solution incorporates temporal patterns, promotional activity, economic indicators, holiday effects, transaction trends, and store-level metadata to improve forecasting accuracy.

Key business drivers considered include:

- Promotional campaigns (`onpromotion`)
- Oil price fluctuations (`dcoilwtico`)
- National and local holidays
- Store transaction volumes
- Store characteristics such as city, state, type, and cluster

The final solution combines XGBoost and CatBoost predictions through model ensembling, producing strong forecasting performance on the validation dataset.

## Feature Engineering

A comprehensive feature engineering pipeline was implemented to capture temporal patterns, seasonality, and external influences on sales performance.

### Time-Based Features

- Year
- Month
- Day
- Day of Week
- Week of Year
- Quarter
- Weekend Indicator

### Cyclical Encodings

To preserve periodic relationships:

- Day of Year (Sine/Cosine)
- Month (Sine/Cosine)
- Day of Week (Sine/Cosine)

### Lag Features

Historical sales patterns:

- Sales Lag 1 Day
- Sales Lag 7 Days
- Sales Lag 14 Days
- Sales Lag 21 Days
- Sales Lag 28 Days

Transaction history:

- Transaction Lag 1 Day
- Transaction Lag 7 Days
- Transaction Lag 14 Days

### Rolling Statistics

Rolling windows were created to capture short-term and long-term trends:

- Rolling Mean (7, 14, 28, 60 Days)
- Rolling Standard Deviation (7, 14, 28, 60 Days)

### Promotion Features

- Current Promotion Count
- 14-Day Promotion Ratio

### External Variables

#### Oil Prices

- Forward Fill
- Backward Fill
- Median Imputation

#### Holiday Features

- Holiday Indicator
- Number of Holidays per Date

### Categorical Encoding

Label encoding applied to:

- Product Family
- City
- State
- Store Type
- Store Cluster

## Model Development

Two gradient boosting models were trained independently and combined through ensembling.

### XGBoost

```python
n_estimators = 500
learning_rate = 0.05
max_depth = 8
early_stopping_rounds = 50
```

### CatBoost

```python
iterations = 500
learning_rate = 0.05
depth = 8
early_stopping_rounds = 50
```

### Validation Strategy

A time-aware validation approach was used:

- Training Set: Historical observations
- Validation Set: Last 30 days of training data

This prevents data leakage and better simulates real-world forecasting scenarios.

### Ensemble Strategy

Final predictions are generated by averaging the outputs of:

- XGBoost
- CatBoost

Predictions are clipped to non-negative values to maintain realistic sales forecasts.

## Model Performance

### Root Mean Squared Logarithmic Error (RMSLE)

| Model | Train RMSLE | Validation RMSLE |
|---------|-------------|------------------|
| XGBoost | 0.5804 | 0.4491 |
| CatBoost | 0.9208 | 0.6839 |
| Ensemble | 0.7624 | 0.5563 |

### Mean Absolute Error (MAE)

| Model | Train MAE | Validation MAE |
|---------|-----------|----------------|
| XGBoost | 30.83 | 46.02 |
| CatBoost | 41.21 | 52.97 |
| Ensemble | 34.54 | 47.92 |

The ensemble model improves overall stability and generalization while reducing sensitivity to individual model weaknesses.
